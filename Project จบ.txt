	-คำภามจากคณะกรรมการ

	. คำถามเกี่ยวกับการวางแผนและการออกแบบ
"การดึงข้อมูลจากเว็บไซต์หรือ API จะทำอย่างไรให้มั่นใจได้ว่าข้อมูลที่ได้รับมีความถูกต้องและเชื่อถือได้?"

คำตอบ: เราจะเลือกแหล่งข้อมูลที่มีชื่อเสียงและได้รับการยอมรับในวงการ เช่น API จากองค์กรหรือหน่วยงานที่มีความน่าเชื่อถือ เช่น ธนาคารหรือองค์กรรัฐบาลที่เผยแพร่ข้อมูลการเงิน และเราจะทำการตรวจสอบความถูกต้องของข้อมูลด้วยการตรวจสอบความสมบูรณ์และความต่อเนื่องของข้อมูลจากแหล่งต่างๆ โดยใช้การทดสอบและการเปรียบเทียบข้อมูลจากแหล่งอื่นๆ
"คุณจะรับประกันได้อย่างไรว่า Dashboard จะแสดงข้อมูลแบบ Real-Time ตามที่ต้องการ?"

คำตอบ: ระบบจะเชื่อมต่อกับ API หรือเว็บไซต์ที่ให้ข้อมูลในรูปแบบเรียลไทม์ โดยจะใช้เทคนิคการดึงข้อมูลทุกๆ ระยะเวลา เช่น ทุก 5 นาที หรือ 10 นาที ขึ้นอยู่กับประเภทของข้อมูล โดยสามารถใช้เทคนิคเช่น WebSocket หรือ Polling เพื่ออัปเดตข้อมูลใน Dashboard โดยอัตโนมัติ
"ข้อมูลจากหลายองค์กรจะถูกแสดงพร้อมกันใน Dashboard ได้อย่างไร? มีข้อจำกัดในการแสดงผลหรือไม่?"

คำตอบ: เราจะใช้ฟีเจอร์ในการกรองข้อมูลใน Dashboard เพื่อให้ผู้ใช้สามารถเลือกดูข้อมูลจากหลายองค์กรได้ในเวลาเดียวกัน โดยการแสดงผลจะมีการแบ่งเป็นแผงข้อมูลหรือกราฟต่างๆ ที่แสดงข้อมูลแยกกันตามองค์กรเพื่อให้สามารถเปรียบเทียบกันได้ ข้อจำกัดที่อาจจะเกิดขึ้นคือความสามารถของเครื่องที่ใช้งาน หรือจำนวนข้อมูลที่ต้องจัดการ แต่จะมีการออกแบบระบบให้สามารถรองรับข้อมูลในปริมาณที่พอเหมาะ



	. คำถามเกี่ยวกับการวิเคราะห์และ AI
"ทำไมคุณถึงเลือกใช้โมเดล AI แบบนี้ในการพยากรณ์การเติบโตของงบการเงิน? โมเดลนี้มีข้อจำกัดอะไรบ้าง?"

คำตอบ: เราเลือกใช้โมเดล ARIMA (AutoRegressive Integrated Moving Average) หรือ Prophet เพราะเป็นโมเดลที่เหมาะกับการพยากรณ์ข้อมูลตามเวลา (Time Series Forecasting) ซึ่งจะทำให้เราคาดการณ์การเติบโตในอนาคตได้ดี โมเดลเหล่านี้มักจะมีข้อจำกัดในการจัดการกับข้อมูลที่มีความผันผวนสูงหรือข้อมูลที่มีความผิดปกติ เช่น ข้อมูลที่ขาดหาย
"คุณจะรับมือกับความไม่แน่นอนในข้อมูลอย่างไร? เช่น ความผิดปกติหรือข้อมูลขาดหาย?"

คำตอบ: เราจะใช้เทคนิคการทำ Data Imputation เพื่อเติมเต็มข้อมูลที่ขาดหาย และทำการตรวจสอบค่าผิดปกติ (Outliers) โดยการวิเคราะห์ข้อมูลที่มีความผิดปกติออกและพิจารณาว่าจะจัดการอย่างไร เช่น การใช้ค่ามัธยฐานหรือค่าความถี่ในการแทนที่
"การพยากรณ์ที่สร้างขึ้นมีความแม่นยำแค่ไหน? อะไรคือการทดสอบและการวัดความแม่นยำของโมเดล?"

คำตอบ: เราจะใช้ Metrics ต่างๆ เช่น Mean Absolute Error (MAE) หรือ Root Mean Squared Error (RMSE) เพื่อวัดความแม่นยำของโมเดล พยายามปรับแต่งโมเดลจนกว่าจะได้ค่าความแม่นยำที่ยอมรับได้
"ในกรณีที่โมเดลพยากรณ์ผิดพลาดหรือไม่แม่นยำในบางช่วงเวลา คุณจะปรับปรุงหรือแก้ไขอย่างไร?"

คำตอบ: ถ้าพบว่าโมเดลไม่แม่นยำในบางช่วงเวลา เราจะทำการ ปรับพารามิเตอร์ หรือ เปลี่ยนเทคนิค เช่น ลองใช้โมเดลใหม่หรือเพิ่มข้อมูลเพิ่มเติมเพื่อลดความผิดพลาดในการพยากรณ์



	. คำถามเกี่ยวกับเทคโนโลยีและเครื่องมือ
"ทำไมคุณถึงเลือกใช้เครื่องมือหรือเทคโนโลยีนี้ในการพัฒนา? เช่น Power BI, Tableau หรือ Python?"

คำตอบ: เราเลือกใช้ Python เนื่องจากมีเครื่องมือที่หลากหลายในการดึงข้อมูลจาก API หรือเว็บไซต์ เช่น BeautifulSoup และ Requests รวมถึงการวิเคราะห์ข้อมูลด้วยเครื่องมืออย่าง Pandas และ NumPy ขณะที่ Power BI หรือ Tableau เหมาะสำหรับการแสดงผลข้อมูลใน Dashboard ที่มีความสวยงามและใช้งานง่าย
"การเชื่อมต่อข้อมูลจากหลายแหล่ง (API, Web Scraping, ฐานข้อมูล) จะทำอย่างไรให้ข้อมูลไม่ขัดแย้งกันหรือมีความขัดแย้ง?"

คำตอบ: เราจะจัดการข้อมูลจากหลายแหล่งด้วยการทำ Data Transformation และ Data Normalization ให้ข้อมูลทั้งหมดอยู่ในรูปแบบเดียวกัน เช่น การแปลงค่าให้ตรงกับหน่วยหรือฟอร์แมตเดียวกันก่อนนำมาใช้ในการวิเคราะห์
"ระบบจะสามารถรองรับการเพิ่มข้อมูลใหม่ๆ ได้หรือไม่? และระบบมีความยืดหยุ่นในการขยายตัวอย่างไร?"

คำตอบ: ระบบจะมีการออกแบบที่สามารถรองรับการเพิ่มข้อมูลใหม่ๆ ได้อย่างง่ายดาย เช่น การเพิ่มแหล่งข้อมูลใหม่ หรือการเพิ่มฟีเจอร์ใหม่ใน Dashboard โดยการใช้ Microservices Architecture ที่ช่วยให้ระบบมีความยืดหยุ่นและสามารถขยายตัวได้ในอนาคต



	. คำถามเกี่ยวกับการรักษาความปลอดภัยและความเป็นส่วนตัว
"ข้อมูลทางการเงินที่ระบบจะจัดเก็บมีความสำคัญมาก คุณมีมาตรการรักษาความปลอดภัยของข้อมูลเหล่านี้อย่างไร?"

คำตอบ: เราจะใช้การ เข้ารหัสข้อมูล (Encryption) ทั้งในการส่งข้อมูลและการเก็บข้อมูล รวมถึงการใช้ ระบบการตรวจสอบสิทธิ์ (Authentication) เช่น OAuth หรือ JWT เพื่อป้องกันไม่ให้บุคคลที่ไม่เกี่ยวข้องเข้าถึงข้อมูล
"คุณจะรับประกันได้อย่างไรว่าไม่มีการละเมิดข้อมูลส่วนบุคคลจากผู้ใช้งาน?"

คำตอบ: เราจะกำหนด สิทธิ์การเข้าถึงข้อมูล และ Role-Based Access Control (RBAC) เพื่อให้แน่ใจว่าผู้ใช้แต่ละคนจะเข้าถึงข้อมูลได้ตามสิทธิ์ที่กำหนดไว้
"หากองค์กรต้องการเก็บข้อมูลเพิ่มเติมที่ละเอียดขึ้น คุณจะรองรับข้อมูลที่เพิ่มขึ้นได้อย่างไรในระยะยาว?"

คำตอบ: เราจะออกแบบฐานข้อมูลให้มีความยืดหยุ่นและสามารถรองรับข้อมูลที่เพิ่มขึ้นได้ โดยใช้ Database Schema ที่สามารถขยายตัวได้ง่าย



	. คำถามเกี่ยวกับการทดสอบและการบำรุงรักษา
"ในระยะยาวคุณมีแผนที่จะอัปเดตหรือปรับปรุงระบบนี้อย่างไร? เช่น การเพิ่มฟีเจอร์ใหม่ หรือการปรับปรุงโมเดลพยากรณ์?"

คำตอบ: เรามีแผนที่จะปรับปรุงระบบโดยการเพิ่มฟีเจอร์ใหม่ๆ เช่น การแสดงผลที่ซับซ้อนขึ้น หรือการปรับปรุงโมเดลพยากรณ์โดยการใช้ข้อมูลเพิ่มเติมและเทคนิคใหม่ๆ
"ระบบจะสามารถทำงานได้ต่อเนื่องหรือไม่ในกรณีที่มีการขัดข้องหรือระบบล่ม?"

คำตอบ: ระบบจะมี ระบบสำรอง (Backup System) และ การฟื้นฟูระบบ (Recovery Plan) ในกรณีที่เกิดข้อผิดพลาดหรือระบบล่ม
"คุณจะทดสอบและปรับปรุงโมเดล AI ได้อย่างไรเมื่อข้อมูลใหม่ๆ เข้ามา?"

คำตอบ: เราจะทำ การทดสอบกับข้อมูลใหม่ อย่างสม่ำเสมอ และปรับปรุงโมเดลโดยใช้ข้อมูลที่เพิ่มเข้ามาเพื่อลดข้อผิดพลาดและเพิ่มความแม่นยำ




	 คำถามเกี่ยวกับประโยชน์และผลลัพธ์
"การพยากรณ์การเติบโตของงบการเงินจะช่วยให้องค์กรใช้ประโยชน์จากข้อมูลนี้อย่างไร? และคุณเห็นว่าอะไรคือผลลัพธ์ที่สำคัญที่สุดจากการใช้ระบบนี้?"

การพยากรณ์จะช่วยให้องค์กรวางแผนการเงินในอนาคตได้ดีขึ้น เช่น การจัดสรรงบประมาณและการตัดสินใจลงทุน ผลลัพธ์สำคัญคือการเพิ่มความแม่นยำในการวางแผนและการจัดการความเสี่ยง
"คุณคาดหวังว่าองค์กรจะใช้งาน Dashboard นี้อย่างไร? และมีการฝึกอบรมผู้ใช้งานอย่างไร?"

องค์กรจะใช้ Dashboard ในการตรวจสอบและวิเคราะห์การเงินแบบ Real-Time พร้อมการฝึกอบรมเบื้องต้นเกี่ยวกับการใช้งานฟีเจอร์ต่างๆ ของระบบ
"ระบบนี้สามารถช่วยในการตัดสินใจเชิงกลยุทธ์ได้อย่างไร? และอะไรคือความแตกต่างจากเครื่องมือการวิเคราะห์การเงินทั่วไป?"

ระบบช่วยในการตัดสินใจเชิงกลยุทธ์ด้วยการให้ข้อมูลเชิงลึกจากการพยากรณ์การเงินที่ช่วยในการวางแผน ส่วนความแตกต่างจากเครื่องมือทั่วไปคือสามารถแสดงข้อมูล Real-Time และเชื่อมโยงข้อมูลจากหลายแหล่งได้ทันที
